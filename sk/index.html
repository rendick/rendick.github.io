1.0 sk functionality

Here you will be introduced to and learn the basic commands of the sk version control system.

1.1 How to use the commands properly?

sk has only 11 commands to make it more user-friendly: init, add, commit, push, rm, clean, info, logs, clone, help, version

init: initialize the project.

add: add the contents of the needed files and directories to the change file

commit: write changes to the repository.

push: update the current repository.

rm: remove files from the working tree.

clean: clean unnecessary sk files.

info: output of information about the current sk project.

clone: clone the repository to a specific directory.

help: display the help menu.

version: display the sk version.

2.0 skignore

skignore is a powerful tool for sk, similar to .gitignore for selecting files & directories that should not to be in the main repository.

2.1 How to use it?

First of all, I need to mention, that .skignore has own template. 

[skignore]
ignored = ["test.txt", "test_dir/test2.txt"]

If you want .skignore to work, you should create it in the main (first) layer of the repository. (like .gitignore)

Put specific files and/or directories into ignored = [ ... ], and it will filter files/directories!

(!) Do not write ./ & ~ prefixes in the ignored line. It won't work, but soon I will fix this problem!

3.0 sk server

In my mind, a sk server would be an "alternative" to services like Github, Gitlab, Gitea, etc to make usability more comfortable.

3.1 How to start it?

You just need to have already installed npm and Node.js in your system.

After making sure you already have installed npm and Node.js and also created sk repository, go to the main/first layer of the repository (directory) and write npm install && npm start
